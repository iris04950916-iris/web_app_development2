# 食譜收藏夾系統 PRD (產品需求文件)

## 1. 專案概述

### 背景與動機
現代人越來越注重健康與自煮生活，但在網路上看到的食譜散落各處，或是手抄的食譜容易遺失。為了解決這個問題，我們需要一個集中管理的「食譜收藏夾」系統，讓使用者可以輕鬆地記錄、分類並快速查找自己喜愛的食譜。

### 目標用戶
熱愛烹飪的家庭煮夫/婦、租屋自炊族、料理愛好者，以及希望有系統地管理飲食與營養資訊的健康注重者。

### 核心價值主張
提供一個簡單、直覺的平台，讓使用者能一站式新增食譜、記錄詳細的材料與步驟，並透過分類、評分系統與營養資訊顯示，幫助他們更有效率地規劃每一餐。

## 2. 功能需求

### 1. 新增食譜
- **使用者故事**：作為一名料理愛好者，我希望能夠新增我的專屬食譜，以便把我的料理秘方記錄下來。
- **功能描述**：提供表單讓使用者輸入食譜名稱、簡介及上傳圖片。

### 2. 列出材料與步驟
- **使用者故事**：作為一名自炊族，我希望在查看食譜時能清楚看到所需的材料清單與詳細步驟，以便我能照著步驟順利完成料理。
- **功能描述**：食譜內頁能條列式呈現材料（名稱與份量）與料理步驟（文字與順序）。

### 3. 食譜分類功能
- **使用者故事**：作為一名家庭主婦，我希望能夠將食譜分類（如：中式、西式、甜點、素食），以便我在規劃不同餐點時能快速找到合適的食譜。
- **功能描述**：支援建立分類標籤，並能透過標籤篩選或搜尋食譜。

### 4. 評分與評論功能
- **使用者故事**：作為一名美食分享者，我希望能夠對做過的食譜給予評分並留下評論，以便記錄這次料理的心得並作為下次改進的參考。
- **功能描述**：使用者可對單一食譜留下 1 到 5 星的評分及文字評論。

### 5. 營養資訊顯示
- **使用者故事**：作為一名注重健康的飲食者，我希望能夠在食譜中看到營養資訊（如熱量、蛋白質、碳水化合物等），以便控管我的飲食攝取。
- **功能描述**：食譜詳細頁面中能提供該道料理的營養素估算數值區塊。

## 3. 非功能需求

### 技術限制
- **後端框架**：Flask
- **模板引擎**：Jinja2
- **資料庫**：SQLite

### 效能與安全考量
- **效能**：頁面載入時間需小於 2 秒，資料庫查詢需優化以確保在食譜數量增加時的檢索速度。
- **安全**：需防範 SQL Injection 與 XSS 攻擊，確保使用者輸入的評論與食譜內容經過適當的過濾與跳脫處理。

## 4. MVP 範圍

### Must Have (必須具備)
- 新增、編輯、刪除食譜功能。
- 列出食譜的材料與步驟。
- 食譜分類功能與基礎的分類篩選。

### Should Have (應該具備)
- 評分與評論功能（可單純記錄自己的心得）。
- 基礎的營養資訊記錄（使用者自行輸入熱量等資訊）。

### Nice to Have (可以有，未來擴充)
- 自動計算營養資訊（串接外部 API）。
- 食譜社群分享功能（公開食譜讓其他人收藏）。
